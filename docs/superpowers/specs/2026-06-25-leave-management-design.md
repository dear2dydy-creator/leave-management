# 직원 연월차 관리 시스템 설계

## 개요

스카이캠프/스카이앤 소속 직원(10~50명)의 연차·월차를 관리하는 내부 웹 도구. 관리자 전용 접근, 복수 관리자 계정 지원.

---

## 기술 스택

- **프레임워크:** Next.js 14 (App Router)
- **ORM:** Prisma
- **DB:** Neon Postgres (Vercel Postgres)
- **인증:** NextAuth.js (Credentials provider, JWT 세션)
- **배포:** Vercel

---

## 부서 및 연차 정책

### 영업부 (월차 방식)
- 대상: 영업지원부 외 전원
- 규칙: 해당 월 만근 시 월차 1일 자동 발생
- 자동 계산되며 관리자가 수동 override 가능

### 영업지원부 (근로기준법 연차)
- 대상: 한민재, 민정후, 최우영, 김인우, 이다영
- 규칙:
  - 입사 1년 미만: 매월 만근 시 1일 발생 (최대 11일)
  - 입사 1년 이후: 15일 기본, 매 2년마다 +1일, 최대 25일
  - 연차 적용기간: 입사일 기준 1년 단위
- 자동 계산되며 관리자가 수동 override 가능

### 지각 규칙 (공통)
- 지각 횟수는 직원별로 **누적** 관리 (연도 리셋 없음)
- 3의 배수 도달 시 자동으로 반차(0.5일) 차감
- 차감 시 휴가 사용내역에 자동 등록 (사유: "지각 3회 차감", 비고: 지각 날짜 목록)

## 데이터 모델

### Employee (직원)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | String (CUID) | PK |
| name | String | 이름 |
| company | Enum | 스카이캠프 / 스카이앤 |
| department | Enum | 영업부 / 영업지원부 |
| position | String | 직급 (팀장, 사원 등) |
| hireDate | DateTime | 입사일 |
| terminationDate | DateTime? | 퇴사일 (null = 재직 중) |
| tardyCount | Int | 누적 지각 횟수 |
| createdAt | DateTime | |

### LeaveBalance (연/월차 잔액 — 적용기간 단위)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | String (CUID) | PK |
| employeeId | String | FK |
| periodStart | DateTime | 적용기간 시작일 |
| periodEnd | DateTime | 적용기간 종료일 |
| allocatedDays | Float | 발생 연/월차 (자동 계산값) |
| bonusDays | Float | 추가일수 (수동) |
| deductionDays | Float | 공제일수 (수동) |
| usedDays | Float | 사용일수 (LeaveRecord 합계) |
| note | String? | 비고 |

잔여일수 = allocatedDays + bonusDays - deductionDays - usedDays (계산 필드)

allocatedDays는 자동 계산값이 기본이며, 관리자가 수동 입력 시 override 값으로 저장됨.
LeaveBalance는 입사일 기준 1년 단위 적용기간마다 1개 레코드 생성.

### LeaveRecord (휴가 사용내역)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | String (CUID) | PK |
| employeeId | String | FK |
| startDate | DateTime | 시작일 |
| endDate | DateTime | 종료일 |
| category | Enum | 연차/월차/반차/병가/공가/결근/기타 |
| days | Float | 사용일수 (반차 = 0.5) |
| reason | String? | 사유 |
| note | String? | 비고 |
| isAutoGenerated | Boolean | 지각 자동 차감 여부 |
| createdAt | DateTime | |

병가/공가/결근/기타는 연차 잔액 차감 없이 기록만 남김.

### AdminUser (관리자)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | String (CUID) | PK |
| name | String | 이름 |
| email | String | 이메일 (로그인 ID) |
| passwordHash | String | bcrypt 해시 |
| createdAt | DateTime | |

## 화면 구성

### 1. 로그인 (`/login`)
- 이메일 + 비밀번호 로그인
- NextAuth Credentials provider

### 2. 대시보드 (`/`)
- 전체 직원 현황 테이블
  - 이름, 회사, 부서, 직급, 적용기간, 발생일수, 사용일수, 잔여일수
  - 퇴사자 행 별도 표시 (회색 처리)
- 직원 추가 버튼
- 엑셀 내보내기 버튼

### 3. 직원 상세 (`/employees/[id]`)
- 기본정보 섹션: 이름, 회사, 부서, 직급, 입사일, 퇴사일, 누적 지각 횟수
- 연/월차 요약: 적용기간, 발생/추가/공제/사용/잔여 (수정 가능)
- 지각 기록 버튼 → 지각 날짜 입력, 3회 차감 자동 처리
- 휴가 사용내역 테이블: 추가/수정/삭제
- 직원 정보 수정, 퇴사 처리 버튼

### 4. 달력 뷰 (`/calendar`)
- 월별 달력
- 전체 직원 휴가 표시 (이름 + 구분)
- 구분별 색상 구분 (연차/월차/반차/병가 등)
- 월 이동 (이전/다음)

### 5. 관리자 설정 (`/settings`)
- 관리자 계정 목록
- 관리자 추가 / 비밀번호 변경 / 삭제

## 연차 자동 계산 로직

### 영업지원부 (근로기준법)
```
입사일로부터 경과 월 수 계산
- 경과 개월 < 12: min(경과 개월, 11)일
- 경과 개월 >= 12:
  근속 연수 = floor(경과 개월 / 12)
  기본 = 15일
  추가 = floor((근속 연수 - 1) / 2) → 최대 10일
  발생 = min(15 + 추가, 25)일
```

### 영업부 (월차)
```
월차 발생 기준: 입사일 기준 매 1개월 단위
예) 1월 14일 입사 → 2월 14일에 1일 발생, 3월 14일에 1일 발생, ...

만근 기준: 입사일~다음 입사일 기념일 사이 결근(LeaveRecord.category = 결근) 기록이 없으면 만근
병가/공가는 만근 여부에 영향 없음
발생일수는 화면 진입 시 실시간 계산 (DB에는 override 값만 저장)
```
